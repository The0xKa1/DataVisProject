# DataVisProject Documentation

This directory is the working documentation hub for the course final project.

## Project Goal

Build a reproducible visual analytics system for a self-selected dataset. The system should support multi-view exploration, coordinated interactions, and clear analysis tasks that the team can explain during the final presentation.

## Course Requirement Snapshot

- Team size: 2-3 members.
- Dataset: not identical to ChinaVis / VAST challenge datasets; not a toy teaching dataset such as iris, titanic, or tips.
- Data complexity: preferably at least 10,000 records, or with temporal, spatial, hierarchical, or multi-entity relational structure.
- Required documentation: division of work, data source and license, AI usage statement, design rationale, reproducible setup, and 1-2 case studies.
- Required explainability: the team must be able to justify analysis tasks, view choices, coordinated interactions, and key design decisions.

## Document Map

- `CURRENT.md`: current project status and immediate focus.
- `TODO.md`: task list with owners, modules, goals, and acceptance criteria.
- `DATASET.md`: data source, license, scale, fields, and processing notes.
- `DESIGN.md`: analysis tasks, visualization design, interaction rationale, references, and original contributions.
- `DIVISION.md`: member responsibilities and presentation ownership.
- `AI_USAGE.md`: AI tool usage statement and team reflection.
- `CASE_STUDIES.md`: 1-2 analysis stories discovered through the system.
- `handoff/`: session handoff records.
- `开发记录/`: personal development records by member.

## Working Rule

Before starting a task, read `CURRENT.md` and `TODO.md`. After finishing a meaningful task, update both files and add a handoff when the work should be passed to another teammate or future agent.
